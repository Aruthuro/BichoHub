## Padrões de implementação

- Camel case para nome de classe e função;
- Documentar parâmetros e retorno de funções;
- Nome de variáveis deve seguir o padrão snake_case, com nome semântico.

## Eventos da semana

Como estamos usando SCRUM, aqui está o cronograma de eventos de uma sprint:

- **Sprint**: 1 semana;
- **Sprint planning**: toda terça 13 horas;
- **"Daily"**: Sexta a tarde 13~18;
- **Sprint Review**: Segunda;
- **Sprint Retrospective**: toda terça 13 horas;
